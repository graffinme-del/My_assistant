Почему «веб не открывается», хотя git вроде правильный
=====================================================

1) Git хранит «снимок» файлов в коммите. На ДИСКЕ могут лежать ДРУГИЕ файлы (старый docker-compose,
   удалённая папка infra). Тогда: git log нормальный, а сайт мёртвый.

2) Случайные порты вроде 34289 вместо 8080 = поднят СТАРЫЙ compose с ${APP_PORT} и без .env.

3) Что затирает файлы: второй деплой (SFTP, rsync с ПК, другой workflow), правка в IDE по SSH.
   Не копируйте проект поверх /opt/my_assistant вручную.

ОДНА команда «привести диск = как на GitHub и поднять контейнеры»:
  cd /opt/my_assistant && bash infra/deploy_on_server.sh

Скрипт сам делает: git reset --hard, .env, docker compose, проверку :8000 и :8080.

Открывать в браузере: http://ВАШ_IP:8080  (не 342xx).
