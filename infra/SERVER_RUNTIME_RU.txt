Прод сейчас: образы api/web/worker в ghcr.io, на сервере .env + runtime.compose.yml в корне DEPLOY_ROOT (compose качается с raw как /runtime.compose.yml).

1) После первого зелёного GitHub Actions → Deploy на сервере уже подтянутся образы. Вручную:
   curl -fsSL https://raw.githubusercontent.com/graffinme-del/My_assistant/main/infra/deploy_on_server.sh | bash

2) Если docker pull пишет 403/денаед — пакеты GHCR сделайте Public:
   GitHub → Packages → каждый пакет api/web/worker → Package settings → Change visibility → Public
   Либо на сервере: docker login ghcr.io (PAT с read:packages).

3) Локальная разработка по-прежнему: docker compose с infra/compose.prod.yml и --project-directory .

4) Не заливайте старый код поверх /opt — он больше не участвует в запуске, но может путать вас; .env не перезаписывайте случайно.

5) Веб открывайте как http://IP:8080/ — API идёт через тот же хост в /api/ (прокси nginx→api). В фаерволе Hetzner достаточно открыть 8080 (и при необходимости 80/443); 8000 снаружи не обязателен. Если раньше в браузере сохранялся apiBase на :8000 — в DevTools → Application → Local Storage удалите ключ apiBase или сделайте жёсткое обновление после деплоя нового образа web.

6) Секреты: .env.local (chmod 600) по желанию — см. .env.local.example. Деплой не трогает ни .env, ни .env.local; ensure_env больше не дописывает строки из .env.example при каждом запуске (это могло давать вторую пустую OPENAI_API_KEY= и «ключ не задан»). Перед деплоем создаётся .env.bak — откат: cp .env.bak .env

7) Не программисту: куда смотреть браузером и какие команды вводить — см. infra/CHECK_WEB_RU.txt. Не добавляйте в команду docker compose флаг --env-file .env.local, если файла .env.local нет (будет ошибка «couldn't find env file»).
