# Быстрый старт (простыми шагами)

## 1. Что уже готово

- Сервер API с делами, событиями, задачами, документами, поиском и сводкой.
- Веб-страница-заглушка.
- Docker Compose для локального запуска и Hetzner.
- GitHub Actions: CI и автодеплой.

## 2. Что нужно заполнить в GitHub Secrets

- `HETZNER_HOST`
- `HETZNER_USER`
- `HETZNER_SSH_KEY`
- `DEPLOY_PATH`

## 3. Как запускать локально

1. Скопировать `env.example` в `.env`.
2. Заполнить пароли и токены.
3. Запустить `docker compose up --build`.
4. Открыть:
   - `http://localhost:8080`
   - `http://localhost:8000/docs`

## 4. Как пользоваться API (минимум)

Во все запросы передавать заголовок:

- `X-API-Token: owner-dev-token` (или свой токен из `.env`)

### Создать дело

`POST /cases`

```json
{
  "title": "Grimme vs Банк",
  "court_name": "Арбитражный суд",
  "case_number": "А40-12345/2026",
  "status": "in_progress",
  "stage": "hearing"
}
```

### Добавить текст после заседания

`POST /cases/{id}/hearing-note`

```json
{
  "text": "Отложение на 19.05. Суд попросил письменно изложить позицию..."
}
```

Система:
- добавит событие в таймлайн,
- попробует извлечь дату заседания,
- создаст задачи по ключевым фразам.
